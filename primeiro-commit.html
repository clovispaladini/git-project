Os principais estágios de um commit:

Um arquivo comitado é um arquivo que efetivamente entrou no controle de versão, porém para que o arquivo possua este status, é necessário verificar seu estágio, que poderá ser:

  1. Untracked files;
  2. Changes to be committed;
  3. Nothing to commit / committed;
  4. Changes not staged for commit;

Verificar o status de um arquivo

O comando git status retorna uma lista indicando em qual estágio cada arquivo do diretório está.

Untracked files

Continuando o exemplo anterior, vamos listar os arquivos do diretório e verificar seu status.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ ls
    index.html  

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master

    No commits yet

    Untracked files:
        (use "git add <file>..." to include in what will be committed)
            index.html

    nothing added to commit but untracked files present (use "git add" to track)

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $

Acima verificamos que o arquivo 'index.html' está no primeiro estario, 'Untracked file', isto é, este arquivo ainda não foi adcionado ao controle de versão, ele esta dentro do diretório onde foi inicializado o Git porem não esta sob o comando do controle de versão.

Changes to be committed

Para que o arquivo começe a fazer parte do controle de versão usaremos o comando git add index.html. Obs: Para adicionar vários arquivos de uma só vez, usar o comando git add ..

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git add index.html

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $

Agora o arquivo esta fazendo parte do controle de versão, e novamente usaremos o comando git status e veremos que o arquivo passou para o segundo estágio, onde esta pronto para ser comitado.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master

    No commits yet

    Changes to be committed:
        (use "git rm --cached <file>..." to unstage)
            new file: index.html

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $
Nothing to commit / committed

Um arquivo comitado é um arquivo que entra efetivamente no controle de versao, onde no momento do commit o Git gera uma identificação desse commit e de quando este arquivo foi comitado.

O comando git commit -m "Meu primeiro commit" irá comitar o arquivo, finalizando o processo de 'commit' e salvando as informações de 'log' deste arquivo.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git commit -m "Meu primeiro commit"
    [master (root-commit) 0cab48c] Meu primeiro commit
     1 file changed, 10 insertions(+)
     create mode 100644 index.html

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $

Agora o arquivo esta comitado, com git status veremos que o repositório não tem mais arquivos fora do sistema de versão.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master
    nothing to commit, working tree clean

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $
Changes not staged for commit

Quando um arquivo que esta sob o controle de versão e seu conteúdo é alterado, como a inserção de uma linha de código por exemplo, este arquivo tem que ser comitado novamente e irá passar pelos mesmos processos listados acima.

No arquivo do exemplo (index.html) foi realizada a alteração de um ponto final para ponto de exclamação em um parágrafo na frase. Novamente o comando git status irá retornar o statos do repositório.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master
    Changes not staged for commit:
        (use "git add <file>..." to update what will be committed)
        (use "git restore <file>..." to discard changes in working directory)
            modified: index.html

    no changes added to commit (use "git add" and/or "git commit -a")

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $

Acima, o Git esta informando que o arquivo foi modificado, reparem que ele nos da a possibilidade de adicionar este arquivo ao controle de versão com o comando git add <file>..., ou tambem podemos voltar este arquivo a versão anterior, antes da alteração quando ele estava comitado, com o comando git restore <file>....

Como a intenção é comitar novamente este arquivo que foi alterado, usaremos os comandos que já vimos anteriormente para adcionar o arquivo no sistema de versão e comitar este mesmo arquivo.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git add index.html
    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master
    Changes to be committed:
        (use "git restore --staged <file>..." to unstage)
            modified: index.html
    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git commit -m "Alterado ponto no final da frase no arquivo index.html"
    [master 15dfdd3] Alterado ponto no final da frase no arquivo index.html
     1 file changed, 1 insertion(+), 1 deletion(-)

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git status
    On branch master
    nothing to commit, working tree clean

Agora o arquivo foi novamente comitado e o Git esta informando que não tem mais nenhum arquivo pendente no sistema de versão

Então vimos os primcipais estágios de um commit no Git e para finalizar esta parte e entendermos melhor os commits, vamos usar o comando init log e visualizar os logs dos commits que efetuamos.

    
    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $ git log
    commit 15dfdd3d1ae85895a7dae9fd8198aa59d63bd100 (HEAD -> master)
    Author: Fernando Machado <seu.email@gmail.com>
    Date:   Mon Sep 01 12:59:47 2020 -0300

        Alterado ponto no final da frase no arquivo index.html

    commit 0cab48c62cb583b6bac1fa177de0f4e0cd71c138
    Author: Fernando Machado <seu.email@gmail.com>
    Date:   Mon Sep 01 12:28:26 2020 -0300

        Meu primeiro commit

    usu@HOST MINGW64 ~/Desktop/git-project (master)
    $

Na informação do 'log' temos no cabeçário um código único que identifica o commit, após temos a identificação do autor do commit, seu email e a data em que o arquivo foi comitado. Por último aparece a mensagem que foi registrada no momento do commit do arquivo.
